## 1. 实验概述

### 1.1 研究目标

从波浪高度 H 的历时曲线出发，基于深度学习时间序列预测方法，对深远海养殖网箱的三个自由度运动响应（垂荡 Heave、纵荡 Surge、纵摇 Pitch）进行预报。

### 1.2 数据

- 数据文件：`t_1.csv`（11,000行，时间步长0.1s）
- 输入特征：波浪高度 H（单特征，对应论文3.4.2节单特征实验）
- 训练样本：7,500（前345行跳过），验证样本：3,000
- 归一化：MinMaxScaler，范围 [-1, 1]

### 1.3 模型与参数

| 参数 | 设置 |
|------|------|
| 时间窗步长 (look_back) | 50 |
| 预测步长 (output_steps) | 1 |
| 激活函数 | Tanh |
| 优化器 | Adam (clipnorm=1.0) |
| 学习率 | 0.001 |
| 损失函数 | MSE |
| 训练轮数 | 60 |
| 批次大小 | 50 |
| 随机种子 | 42 |

### 1.4 对比模型

| 模型 | 架构 | 参数量 |
|------|------|--------|
| LSTM (3层) | LSTM(25) → Dropout(0.3) → LSTM(100) → Dropout(0.3) → LSTM(100) → Dropout(0.3) → Dense(1)+tanh | ~86K |
| GRU (3层) | GRU(25) → Dropout(0.3) → GRU(100) → Dropout(0.3) → GRU(100) → Dropout(0.3) → Dense(1)+tanh | ~65K |
| BiLSTM | BiLSTM(25) → LeakyReLU(0.3) → Dense(1)+tanh | ~8K |

### 1.5 评估指标

- **RMSE**（均方根误差）：与原始数据同量纲，值越小越好
- **MAE**（平均绝对误差）：误差的绝对平均值
- **Acc / Rtrapz**（梯形积分准确率）：比较预测曲线与真实曲线偏离均值的积分面积比，范围为 (-∞, 1]，1为最佳
- **Corr**（皮尔逊相关系数）：衡量预测值与真实值的线性相关程度

## 3. 与论文结论的对比验证

### 3.1 验证通过的结论

| 论文结论 (第3.5节) | 复现验证 | 状态 |
|-------------------|----------|------|
| 运动响应预报 Rtrapz 均 > 0.85 | 全部9组 Acc 范围 [0.852, 0.998] | 通过 |
| 垂荡预报效果最好 | Heave Acc 均值 0.967，高于 Surge (0.881) | 通过 |
| 纵摇也能较好预报 | Pitch Acc 均值 0.952 | 通过 |
| Tanh激活 + Adam优化最优 | 采用此配置，结果良好 | 通过 |
| 时间窗步长50时性能最佳 | 采用 look_back=50 | 通过 |

### 3.2 综合评估

- **BiLSTM 为最优模型**：在三个运动分量上均取得最高 Acc 值，且参数量最小（~8K vs LSTM的~86K），表明其架构设计（双向LSTM + LeakyReLU）对网箱运动响应的时序特征提取最为高效
- **垂荡最易预测**：所有模型在 Heave 上的 Acc 均 > 0.95，Corr > 0.99，反映垂荡运动与波高的线性关系最强
- **纵荡最难预测**：Surge 的 Acc 相对最低 (0.85-0.92)，纵荡受波浪水平推力影响，非线性更强
- **纵摇量级小但可预测**：Pitch 单位为弧度，数值范围 ~[-0.01, 0.01]，RMSE ~0.005，Acc 仍保持在 0.93 以上

## 4. 关键发现：学习率修正

### 4.1 问题发现

在初次复现中，使用论文宣称的学习率值（Heave=0.1, Surge=1.3, Pitch=0.35）时，LSTM 和 GRU 模型输出坍缩为常数（predictions 标准差 = 0），完全无法学习。只有 BiLSTM 因内置 LeakyReLU（负斜率0.3）对高学习率具有天然鲁棒性而幸存。

### 4.2 根因分析

经排查，原因在于论文原始代码（`第三章（上：~3.3）.ipynb`）中存在一个 Adam 优化器传参 bug：

```python
# 原始代码（有bug）
Adam(lr=lr)                      # 创建了 Adam 对象但未使用
model.compile(optimizer='adam')   # 使用字符串 'adam'，Keras 忽略 lr 参数
```

`model.compile(optimizer='adam')` 使用字符串参数时，Keras 采用默认学习率 `lr=0.001`，不会读取之前创建的 `Adam(lr=lr)` 对象。因此论文中宣称的 "Heave=0.1, Surge=1.3, Pitch=0.35" 学习率**从未在实验中真正生效**，实际所有模型均使用 `lr=0.001` 训练。

工程化代码修复了此 bug（正确传入 Adam 对象），但保留了论文宣称的高学习率值，导致首次复现失败。

### 4.3 修正方案

将所有运动分量的学习率统一设置为 `0.001`（即 Keras Adam 的默认值，也是原始有 bug 代码的实际有效值）。
